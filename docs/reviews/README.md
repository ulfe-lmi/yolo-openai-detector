# Reviews

Use this directory to archive strategic, cross-model, or human reviews.

Suggested naming:

```text
YYYY-MM-DD-review-N.md
```

A review should include:

- repository commit or PR reviewed;
- scope;
- findings;
- accepted findings;
- rejected findings;
- follow-up work orders;
- evidence required for closure.

Do not use review files as a substitute for tests.
